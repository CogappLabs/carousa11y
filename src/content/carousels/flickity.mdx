## Our Implementation Notes

Flickity applies `aria-hidden` to inactive slides but keeps focusable elements inside. We manage `tabindex` on slide links to prevent focus traps.

### The Focus Trap Problem

Flickity sets `aria-hidden="true"` on non-visible slides, but links and buttons inside those slides remain focusable. This creates a focus trap where screen reader users can tab into content they can't see.

### Our Fix

```javascript
const updateFocusableElements = (selectedIndex) => {
  slides.forEach((slide, i) => {
    const focusables = slide.querySelectorAll('a, button, input, select, textarea')
    focusables.forEach((el) => {
      el.setAttribute('tabindex', i === selectedIndex ? '0' : '-1')
    })
  })
}

flkty.on('select', () => {
  updateFocusableElements(flkty.selectedIndex)
})

// Initial setup
updateFocusableElements(0)
```

### Custom Controls

Built-in page dots are not keyboard accessible. If you include pagination, implement custom buttons with:
- Proper `<button>` elements
- `aria-label="Go to slide N"`
- `aria-current="true"` on active dot
- Focus ring styles
